**What viewProgressRail must do:**

The rail renders one segment per card in `model.cards`. Each segment is a thin colored bar (2px height) that the user can click to navigate to that card. The rail is full width with small gaps between segments.

Segment color states:
- Active (current card, `i == currentIdx`): `Color.orange` (`#F0DFAF`)
- Completed (before current, `i < currentIdx`): `Color.green` (`#429F59`)
- Pending (after current, `i > currentIdx`): `Color.grey` at reduced opacity (`Element.alpha 0.4`)

Each segment must be clickable: `Element.Events.onClick (NavigateTo i)`.

The segments fill the full width proportionally with `Element.fillPortion 1` (equal portions).

The `stap N/M` counter that was on the right of viewTopCheckboxes is REMOVED — it moves to the bottom nav (Task 2). Do not include it here.

Implementation sketch:
```elm
viewProgressRail : Model Msg -> Int -> Element.Element Msg
viewProgressRail model currentIdx =
    let
        totalCards = List.length model.cards

        viewSegment i _ =
            let
                color =
                    if i == currentIdx then
                        Color.orange
                    else if i < currentIdx then
                        Color.green
                    else
                        Color.grey

                attrs =
                    [ Element.width (Element.fillPortion 1)
                    , Element.height (Element.px 3)
                    , Background.color color
                    , Element.Events.onClick (NavigateTo i)
                    , Element.pointer
                    ]
                    ++ (if i > currentIdx then [ Element.alpha 0.35 ] else [])
            in
            Element.el attrs Element.none
    in
    Element.row
        [ Element.width Element.fill
        , Element.spacing 2
        , Element.paddingXY 0 4
        ]
        (List.indexedMap viewSegment model.cards)
```

Add a `viewBottomNav` function to `src/Form/View.elm` and integrate it into `viewCardChrome` using `Element.inFront`.

**Bottom nav layout:** Three-zone row pinned to the bottom of the viewport.
- Left: `< vorige` (prev button, navigates to `currentIdx - 1`)
- Center: current card name + amber `[N]` incomplete count when items remain
- Right: `volgende >` (next button, navigates to `currentIdx + 1`)

**Card names for center label** (derive from the card at `currentIdx`):
- DashboardCard → "overzicht"
- IntroCard _ → "intro"
- GroupMatchesCard _ → "groepen"
- BracketCard _ → "schema"
- TopscorerCard _ → "topscorer"
- ParticipantCard _ → "gegevens"
- SubmitCard → "inzenden"

**Incomplete count `[N]`** (amber, shown when N > 0):
- DashboardCard / IntroCard: no count (always 0)
- GroupMatchesCard: count = number of unscored group matches. Derive as: `List.length model.bet.answers.matches - (List.length (List.filter (\(_, ans) -> Bets.Types.Answer.GroupMatch.isComplete ans) (Dict.toList model.bet.answers.matches)))`. Alternatively use the simpler: if `allGroupsComplete model` then 0 else show `[?]` (use `[!]` if exact count is hard to compute — see discretion note below).
- BracketCard: 0 if `Form.Bracket.isCompleteQualifiers model.bet`, else show `[!]`
- TopscorerCard: 0 if `Form.Topscorer.isComplete model.bet`, else show `[!]`
- ParticipantCard: 0 if `Form.Participant.isComplete model.bet`, else show `[!]`
- SubmitCard: no count

Use discretion: if computing exact counts is complex, showing `[!]` when incomplete and nothing when complete satisfies NAV-03 (the requirement says "amber [N] incomplete count" — `[!]` is acceptable if exact count requires deep domain logic).

**Prev/next disabled states:**
- At first card (`currentIdx == 0`): prev is disabled — `Element.alpha 0.35`, `Element.htmlAttribute (Html.Attributes.style "cursor" "not-allowed")`, no onClick
- At last card (`currentIdx == List.length model.cards - 1`): next is disabled — same treatment
- Otherwise: active — `Element.pointer`, `Element.Events.onClick (NavigateTo target)`, `Element.mouseOver [Font.color Color.activeNav]`

**Hover state:** Use `Element.mouseOver [ Font.color Color.activeNav ]` for active buttons.

**Fixed positioning:** The bottom nav is a fixed overlay. Use `Element.inFront` on the `viewCardChrome` outer column to attach a full-width bar pinned to the bottom. In elm-ui, fixed bottom is achieved with:
```elm
Element.el
    [ Element.width Element.fill
    , Element.alignBottom
    , Background.color (Element.rgb255 0x2B 0x2B 0x2B)  -- #2b2b2b dark background
    , Element.paddingXY 16 0
    , Element.height (Element.px 48)
    , Border.color Color.terminalBorder
    , Border.widthEach { top = 1, bottom = 0, left = 0, right = 0 }
    ]
    (Element.row [ Element.width Element.fill, Element.height Element.fill, Element.centerY ]
        [ prevButton, centerLabel, nextButton ])
```

Attach to `viewCardChrome` by adding `Element.inFront (viewBottomNav model i)` to the `columnAttrs` list.

The existing `Element.paddingEach { bottom = 64 }` in `columnAttrs` already reserves space so content does not hide behind the nav — keep it.

You will need `import Element.Border as Border` if not already imported.

Implementation note: `Element.inFront` places the element in the coordinate space of the parent. Combined with `Element.alignBottom`, it pins to the bottom of the column. This matches the existing pattern used for `viewStatusBar` in View.elm.
