**Part A — displayHeader refactor in `src/UI/Text.elm`:**

The current implementation uses a single `Element.el` with `Element.text ("--- " ++ String.toUpper txt ++ " ---")` which makes all characters the same amber color. The prototype has dashes in `var(--border)` color (#5f5f5f, `Color.terminalBorder`) and the title in `var(--amber)` (#f0dfaf, `Color.orange`).

Replace with a row layout:
```elm
displayHeader : String -> Element.Element msg
displayHeader txt =
    let
        dashStyle =
            [ UI.Font.mono
            , Font.size 10
            , Font.color Color.terminalBorder
            , Font.letterSpacing 1.8
            ]

        titleStyle =
            [ UI.Font.mono
            , Font.size 10
            , Font.color Color.orange
            , Font.letterSpacing 1.8
            ]
    in
    Element.row
        [ Element.width Element.fill
        , Element.paddingXY 0 10
        , Element.spacing 8
        ]
        [ Element.el dashStyle (Element.text "---")
        , Element.el titleStyle (Element.text (String.toUpper txt))
        , Element.el dashStyle (Element.text "---")
        ]
```

Note: The `Style.header2` function is no longer needed by `displayHeader` directly after this change, but it remains in `UI.Style.elm` as it may be called elsewhere via `UI.Style.header2 attrs`. Keep `header2` in Style.elm — just the visual attributes change (Task 1 handles that).

Add `import UI.Color as Color` to `UI.Text.elm` imports if it is not already there (check: `UI.Color` is already imported as `UI.Color`).

**Part B — introduction style in `src/UI/Style.elm`:**

Replace the `introduction` function to match prototype `.dash-intro` + `.dash-intro p`:
```elm
introduction : List (Element.Attribute msg) -> List (Element.Attribute msg)
introduction attrs =
    attrs
        ++ [ UI.Font.mono
           , Font.size 11
           , Font.color Color.grey
           , Element.spacing 12
           , Element.paddingEach { top = 10, bottom = 10, left = 14, right = 14 }
           , Border.widthEach { bottom = 0, top = 0, left = 2, right = 0 }
           , Border.color Color.activeNav
           , Background.color (Element.rgba255 0xF0 0xA0 0x30 0x0A)
           ]
```

Notes:
- `Font.size 11` — literal, prototype says 11px
- `Font.color Color.grey` — grey (#6e6e6e) is the closest to `--text-dim` (#8f8f7f); `Color.grey = rgb255 110 110 110`. Use `Color.grey`.
- `Element.spacing 12` — elm-ui's `spacing` on a paragraph/column approximates line-height; 12px spacing at 11px font gives ~1.75 effective line density
- `Border.widthEach` with left=2 — the 2px orange left border
- `Border.color Color.activeNav` — `activeNav = rgb255 0xF0 0xA0 0x30` matches prototype `--active` (#f0a030)
- `Background.color (Element.rgba255 0xF0 0xA0 0x30 0x0A)` — rgba(240,160,48,0.04) as used in Form/Dashboard.elm
- Add `import Element.Background as Background` to `UI.Style.elm` if not present — it IS already imported (check: yes, `Background.color` is used throughout Style.elm)

After making changes, run `make build` to confirm compilation.
