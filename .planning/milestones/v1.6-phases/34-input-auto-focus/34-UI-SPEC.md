## Interaction Contract

This phase is behavior-only: no new visual components are added. The three auto-focus interactions share a single implementation pattern.

### Pattern: `Browser.Dom.focus` via `Task.attempt`

All three focus targets use the same Elm mechanism:

```
Task.attempt (\_ -> NoOp) (Browser.Dom.focus "element-id")
```

`Browser.Dom` and `Task` are already imported in `Main.elm`. No new imports required.

### Focus Target 1 — Comment Input (FOCUS-01)

**Trigger:** `ShowCommentInput` Msg is dispatched (currently from `Events.onFocus` on the comment trap field).

**Current behavior:** `ShowCommentInput` handler sets `showComment = True`, returns `Cmd.none`. The multiline `commentInput` replaces the trap field in the DOM on next render.

**Required change:** `ShowCommentInput` handler emits `Browser.Dom.focus "comment-input"` as a `Cmd`. The `Input.multiline` in `viewCommentInput` must have `Element.htmlAttribute (Html.Attributes.id "comment-input")` added to its attribute list.

**Visual state after focus:**
- Bottom border of `terminalInput` changes from `Color.terminalBorder` (`#4f4f4f`) to `Color.orange` (`#f0dfaf`) — this is already handled by `Element.focused [ Border.color Color.orange ]` in `UI.Style.terminalInput`.
- No other visual change.

**Timing:** Focus must be issued as a command from the `update` function after `showComment` is set to `True`. The DOM renders the multiline field on the next frame; `Browser.Dom.focus` issued in the same `Cmd.batch` will succeed because elm-ui processes the DOM update before executing commands from the same update cycle.

**Fallback:** `Task.attempt (\_ -> NoOp)` silently ignores `Browser.Dom.NotFound` if the element is not yet in the DOM. No error state needed.

### Focus Target 2 — Participant Name Field (FOCUS-02)

**Trigger:** The `ParticipantCard` becomes the active card, i.e. `model.idx` advances to the index of the `ParticipantCard` in `model.cards`.

**Current behavior:** `NavigateTo page` sets `model.idx = page`, returns `Cmd.none` (plus scroll-to-top).

**Required change:** When `NavigateTo page` is handled and the card at index `page` is `ParticipantCard _`, emit `Browser.Dom.focus "participant-name"` in addition to `Browser.Dom.setViewport 0 0`. The `Input.text` for the name field in `Form/Participant.elm` (`view` function, first field with tag `NameTag`) must have `Element.htmlAttribute (Html.Attributes.id "participant-name")` added.

**Detection:** In the `NavigateTo` handler in `Main.elm`, check `List.drop page model.cards |> List.head` and match on `Just (ParticipantCard _)`.

**Visual state after focus:** The name field prompt char changes from `-` to `>`, and the field bottom border changes to `Color.orange`. This is already handled by the existing `isActive = state.activeField == Just tag` logic — however that logic tracks focus via `FocusField`/`BlurField` Msg, not `Browser.Dom.focus`. After `Browser.Dom.focus`, the browser fires a `focus` event on the input, which triggers `Html.Events.onFocus (FocusField NameTag)` already wired in `Form/Participant.elm`. So the visual state updates automatically via existing code.

**Fallback:** `Task.attempt (\_ -> NoOp)` on focus failure.

### Focus Target 3 — Blog Post Text Area (FOCUS-03)

**Trigger:** `ShowPostInput` Msg is dispatched (currently from `Events.onFocus` on the post trap field).

**Current behavior:** `ShowPostInput` handler sets `showPost = True`, returns `Cmd.none`. The expanded form with `postInput` (multiline) replaces the trap field.

**Required change:** `ShowPostInput` handler emits `Browser.Dom.focus "post-input"`. The `Input.multiline` in `viewPostInput` (`postInput` local function) must have `Element.htmlAttribute (Html.Attributes.id "post-input")` added to its attribute list.

**Visual state after focus:** Same as comment input — bottom border shifts to `Color.orange` via existing `Element.focused` style. No other visual change.

**Fallback:** `Task.attempt (\_ -> NoOp)`.

## Element ID Registry

These HTML IDs must be unique and stable. They are the contract between the `update` function and the DOM.

| ID | Location | Input Type | Focus Trigger |
|----|----------|------------|---------------|
| `comment-input` | `Activities.viewCommentInput` → `commentInput` function | `Input.multiline` | `ShowCommentInput` handler |
| `participant-name` | `Form/Participant.view` → first `inputField` (tag `NameTag`) | `Input.text` | `NavigateTo` handler (when target card is `ParticipantCard`) |
| `post-input` | `Activities.viewPostInput` → `postInput` function | `Input.multiline` | `ShowPostInput` handler |
