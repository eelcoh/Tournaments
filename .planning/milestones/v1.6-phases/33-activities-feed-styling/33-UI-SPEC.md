## Implementation Notes

### Border override pattern
`UI.Style.resultCard` appends `Border.width 1` as a base. To override per-side widths, pass `Border.widthEach` as an override attr **before** the base attrs — elm-ui applies the last declaration, so callers must pass overrides in the attrs list argument to `resultCard`, not after it. Verify the existing `blogBox` pattern already uses this correctly before changing border width.

Actually: `resultCard attrs` is defined as `attrs ++ [ ... base styles ]`. Because attrs come first and base styles come after, base `Border.width 1` will override a `Border.widthEach` in attrs. The executor must verify the attr ordering and may need to restructure `resultCard` call or inline the card attrs directly in `blogBox`/`commentBox` to ensure `Border.widthEach` wins.

Source: CONTEXT.md code context note on `resultCard` override pattern.

### Color.zenGreen naming
CONTEXT.md grants Claude discretion on the constant name. `zenGreen` is the recommended name: it is the most specific (Zenburn color scheme + green), avoids collision with `Color.green`, and matches the project's Zenburn-themed naming convention (`primaryDark`, `terminalBorder`, `terminalAccentDim`).
