<task type="auto">
  <name>Task 2: Fix GroupBoundary height to prevent scroll wheel layout jumps</name>
  <files>src/Form/GroupMatches.elm</files>
  <action>
In `viewScrollItem`, the `GroupBoundary` branch currently renders without a fixed height. This causes a vertical layout jump when the scroll window crosses a group boundary, because boundary rows have no explicit height while `viewScrollLine` rows have `height (px 44)`.

Change the `GroupBoundary` branch in `viewScrollItem` to add `Element.height (Element.px 44)` and `Element.centerY`:

```elm
viewScrollItem : MatchID -> ScrollItem -> Element.Element Msg
viewScrollItem cursor item =
    case item of
        MatchRow matchData ->
            viewScrollLine cursor matchData

        GroupBoundary grp ->
            Element.el
                [ centerX
                , Font.color Color.grey
                , UI.Font.mono
                , Element.height (Element.px 44)
                , centerY
                ]
                (Element.text ("-- " ++ G.toString grp ++ " --"))
```

This is the only change needed in this file. The `centerY` attribute on the outer `el` vertically centers the text within the 44px height. The text itself does not need a separate `centerY` since it is the direct content.

Do NOT change `updateCursor`, `nextMatch`, or any other navigation logic — the research confirmed cursor navigation already correctly skips boundary labels since `allMatchIDs` contains only `MatchID` values, never `GroupBoundary` items.
  </action>
