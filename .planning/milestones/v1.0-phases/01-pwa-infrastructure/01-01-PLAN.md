  <action>
Download the two Sometype Mono variable woff2 files from the Google Fonts GitHub repository (OFL license) and create the fonts.css @font-face stylesheet.

Step 1 — Create the directory:
```
mkdir -p assets/fonts/
```

Step 2 — Download the variable font files:
```
curl -L -o "assets/fonts/SometypeMono[wght].woff2" \
  "https://github.com/googlefonts/sometype-mono/raw/main/fonts/variable/SometypeMono%5Bwght%5D.woff2"

curl -L -o "assets/fonts/SometypeMono-Italic[wght].woff2" \
  "https://github.com/googlefonts/sometype-mono/raw/main/fonts/variable/SometypeMono-Italic%5Bwght%5D.woff2"
```

If the above paths return 404, try the `main` branch path with `Italic`:
```
curl -L -o "assets/fonts/SometypeMono-Italic[wght].woff2" \
  "https://github.com/googlefonts/sometype-mono/raw/main/fonts/variable/SometypeMono-Italic%5Bwght%5D.woff2"
```

Verify both files are non-empty woff2 files (each should be 30KB+). If the GitHub path structure differs, browse https://github.com/googlefonts/sometype-mono/tree/main/fonts/variable to find the exact filenames.

Step 3 — Write assets/fonts/fonts.css:
```css
/* Self-hosted Sometype Mono variable font
   Source: https://github.com/googlefonts/sometype-mono (OFL-1.1 license)
   Replaces Google Fonts remote load — enables app-shell caching in service worker */

@font-face {
  font-family: 'Sometype Mono';
  src: url('SometypeMono[wght].woff2') format('woff2');
  font-weight: 400 700;
  font-style: normal;
  font-display: swap;
}

@font-face {
  font-family: 'Sometype Mono';
  src: url('SometypeMono-Italic[wght].woff2') format('woff2');
  font-weight: 400 700;
  font-style: italic;
  font-display: swap;
}
```

The `url()` paths are relative to the location of fonts.css (i.e. relative to `assets/fonts/`), so no path prefix is needed. The `font-weight: 400 700` range syntax is the correct variable font syntax — do NOT write two separate `@font-face` blocks for 400 and 700 individually.

  <action>
Generate four PNG icons using ImageMagick. The design is orange text "WC26" on a dark background (#0d0d0d), matching the app's terminal aesthetic. No external artwork source exists in the project.

Step 1 — Verify ImageMagick is available:
```
convert --version
```
If not available, install with: `sudo apt-get install imagemagick` or `sudo pacman -S imagemagick`

Step 2 — Generate the icons:

```bash
# Base icon (any purpose) — 512x512: orange text "WC26" on dark background
convert -size 512x512 xc:"#0d0d0d" \
  -fill "#ff8c00" \
  -font "DejaVu-Sans-Bold" \
  -pointsize 140 \
  -gravity Center \
  -annotate 0 "WC26" \
  assets/icon-512.png

# 192x192 version (any purpose) — scale down from the 512
convert assets/icon-512.png -resize 192x192 assets/icon-192.png

# Maskable 512x512 — same design but content must stay within inner 80% safe zone
# (410x410 centered area on a 512x512 canvas). The text at 140pt centered is well within this.
# Use identical design — the dark background fills the full canvas automatically.
cp assets/icon-512.png assets/icon-512-maskable.png

# Apple touch icon — 180x180, same design
convert assets/icon-512.png -resize 180x180 assets/apple-touch-icon.png
```

If the DejaVu-Sans-Bold font name fails, try `-font DejaVuSans-Bold` or omit the `-font` flag (ImageMagick will use its default font). The goal is readable "WC26" text — font choice is secondary to the icon existing and being the right dimensions.

Verify dimensions:
```bash
identify assets/icon-192.png assets/icon-512.png assets/icon-512-maskable.png assets/apple-touch-icon.png
```

Create the Web App Manifest at src/manifest.json. This file is served at /manifest.json after the Makefile copies it to build/ (that copy is handled in Plan 02).

Write src/manifest.json with the following exact content:

```json
{
  "name": "Voetbalpool",
  "short_name": "Voetbalpool",
  "description": "Voorspel de uitslag van WK 2026 wedstrijden",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#0d0d0d",
  "theme_color": "#ff8c00",
  "icons": [
    {
      "src": "assets/icon-192.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "any"
    },
    {
      "src": "assets/icon-512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any"
    },
    {
      "src": "assets/icon-512-maskable.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "maskable"
    }
  ]
}
```

Notes:
- `background_color` is `#0d0d0d` to match the `body` background in index.html
- `theme_color` is `#ff8c00` (orange accent from UI.Style)
- Icon `src` paths are relative to manifest.json at the build root — `assets/icon-192.png` resolves correctly since both manifest.json and assets/ are at the root of build/
- Do NOT include an apple-touch-icon entry in the manifest — iOS ignores manifest icons; the apple-touch-icon link tag in index.html handles iOS (added in Plan 02)
- `purpose: "any"` and `purpose: "maskable"` must be separate icon entries (do not combine as "any maskable" — deprecated pattern)
