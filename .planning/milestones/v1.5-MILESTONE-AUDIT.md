### ⚠ Integration Gap

**INT-01: RefreshTopscorerResults — missing testMode guard**

- **File:** `src/Main.elm` line 928
- **Issue:** `RefreshTopscorerResults` unconditionally calls `Results.Topscorers.fetchTopscorerResults` against the live network. All three peer handlers have `testMode` branches (`RefreshRanking:740`, `RefreshResults:817`, `RefreshKnockoutsResults:904`). `TSResults` is included in the test-mode nav list (`View.elm` line 155 via MODE-04), so the page is reachable.
- **Consequence:** Navigating to `#topscorer` in test mode fires a real HTTP request. In an offline demo context, `model.topscorerResults` stays `NotAsked` or fails, rendering a spinner/error rather than dummy data.
- **Severity:** Tech debt — no v1.5 requirement explicitly covers topscorer results dummy data, but contradicts the milestone goal of "demonstration without a live backend."
- **Fix:** Create `src/TestData/TopscorerResults.elm` with a `dummyTopscorerResults : TopscorerResults` value; add `if model.testMode then` branch to `RefreshTopscorerResults` in `Main.elm`.
