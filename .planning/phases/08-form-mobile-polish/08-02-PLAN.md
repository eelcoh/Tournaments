**Part A — Form/Submit.elm: "verzenden..." text when submission in-flight**

In the `view` function, find the Loading branch:
```elm
( _, Loading, _ ) ->
    ( introSubmitting, submit Inactive NoOp "inzenden" )
```

Change the button text from `"inzenden"` to `"verzenden..."`:
```elm
( _, Loading, _ ) ->
    ( introSubmitting, submit Inactive NoOp "verzenden..." )
```

This makes the in-flight state visually distinct: the button is disabled AND the text changes to acknowledge the action is processing.

**Part B — View.elm: nav button white mouseOver feedback**

In the `viewFormNavBar` function (added in Plan 01), update the nav button element attributes to add `Element.mouseOver [ Font.color Color.white ]` for buttons that are active (not disabled).

For the active (non-greyed) vorige and volgende buttons:
```elm
Element.el
    [ Element.Events.onClick msg
    , Element.pointer
    , Element.height (Element.px 48)
    , Element.centerY
    , Font.color Color.orange
    , UI.Font.mono
    , Element.mouseOver [ Font.color Color.white ]
    ]
    (Element.text label)
```

White is brighter than orange — gives clear visual acknowledgment on desktop hover and on mobile the brief active-state press shows the color change before navigation.

For disabled (greyed) buttons, do not add mouseOver — keep `Font.color Color.grey` only and no `Element.pointer`.

Note: On mobile, elm-ui `mouseOver` maps to CSS `:hover`. The tap briefly triggers the hover state before navigation completes, providing a flash of feedback. This requires no new Msg variants or state changes.
