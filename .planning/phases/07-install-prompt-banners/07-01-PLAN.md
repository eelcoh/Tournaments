<objective>
Wire up the JS <-> Elm bridge for install prompt banners: pass platform/install state as flags at init, capture the BeforeInstallPrompt event in JS, expose ports for triggering install and persisting dismiss state.

Purpose: Banners cannot be shown without knowing (a) whether the app is already installed, (b) whether Android's native install prompt is available, and (c) whether the user has already dismissed. All three come from JS-land.

Output: Ports.elm (new), updated index.html (JS bridge), updated Types.elm (Flags + InstallBannerState + Model + Msg), updated Main.elm (port subscriptions + init derivation).
</objective>

3. **Capture BeforeInstallPrompt** before Elm loads (the event can fire before JS runs):
   ```js
   var deferredPrompt = null;
   window.addEventListener('beforeinstallprompt', function(e) {
     e.preventDefault();
     deferredPrompt = e;
   });
   ```
   Place this listener as early as possible — in a `<script>` block in `<head>` before `main.js`.

4. **Wire Elm ports after init** (in the existing init script block):
   ```js
   // Port: triggerInstall — call the deferred prompt
   app.ports.triggerInstall.subscribe(function() {
     if (deferredPrompt) {
       deferredPrompt.prompt();
       deferredPrompt.userChoice.then(function() {
         deferredPrompt = null;
       });
     }
   });

   // Port: persistDismiss — write dismiss count to localStorage
   app.ports.persistDismiss.subscribe(function(count) {
     localStorage.setItem('installBannerDismissCount', String(count));
   });
   ```

5. The existing service worker registration script stays unchanged.

Note: `hasBeforeInstallPrompt` passed in flags will be `false` at page load because the event fires asynchronously. The banner logic in Elm will need to check a model field updated via a port message when the event actually fires. Add a **fourth port subscription** for this:
   ```js
   window.addEventListener('beforeinstallprompt', function(e) {
     e.preventDefault();
     deferredPrompt = e;
     // Notify Elm that the prompt is available
     if (app && app.ports && app.ports.onBeforeInstallPrompt) {
       app.ports.onBeforeInstallPrompt.send(null);
     }
   });
   ```
   Because `app` may not exist yet when this fires, store the event first and check in a DOMContentLoaded or after init. The cleanest approach: capture in early script, then after `app` is initialized, set up the port bridge:
   ```js
   // After var app = Elm.Main.init(...)
   if (deferredPrompt) {
     // Event already fired before Elm loaded
     app.ports.onBeforeInstallPrompt.send(null);
   }
   // Forward future firings
   window.addEventListener('beforeinstallprompt', function(e) {
     e.preventDefault();
     deferredPrompt = e;
     app.ports.onBeforeInstallPrompt.send(null);
   });
   ```
   Remove the early listener from `<head>` since the late binding handles both cases. Keep the early `var deferredPrompt = null;` declaration in `<head>` so the variable is in scope.

The final index.html structure:
