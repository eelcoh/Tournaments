<objective>
Restructure the group match score input so the keyboard is the primary interaction, with flags always visible in a permanent match header row and a hidden text-input overlay revealed via "andere score".

Purpose: On mobile, keyboard buttons are the fastest path to setting a score. The current layout shows text inputs as the primary interface. This change makes keyboard-first the default, with flags always visible above whichever input body is shown, and text entry accessible via a link for unusual scores not on the keyboard.
Output: Updated Types.elm (new state + msgs) and GroupMatches.elm (split viewInput into viewMatchHeader + viewScoreInputs, flag images always rendered, overlay).

`viewScoreInputs` — only rendered in the overlay branch, contains the score text input fields:

```elm
viewScoreInputs : MatchID -> Maybe Score -> Element.Element Msg
viewScoreInputs matchID mScore =
    let
        makeAction act val =
            case String.toInt val of
                Just v -> act v
                Nothing -> NoOp

        inputField v act =
            Input.text
                (UI.Style.scoreInput
                    [ width (px 60)
                    , Border.rounded 0
                    , Element.htmlAttribute (Html.Attributes.attribute "inputmode" "numeric")
                    ]
                )
                { onChange = makeAction act
                , text = v
                , label = Input.labelHidden ".."
                , placeholder = Just (Input.placeholder [] (Element.text v))
                }

        wrap fld =
            Element.el (UI.Style.wrapper [ centerX, centerY ]) fld

        extractScore extractor =
            mScore
                |> Maybe.andThen extractor
                |> Maybe.map String.fromInt
                |> Maybe.withDefault ""

        homeInput = inputField (extractScore S.homeScore) (UpdateHome matchID) |> wrap
        awayInput = inputField (extractScore S.awayScore) (UpdateAway matchID) |> wrap
    in
    Element.row (UI.Style.activeMatch [ centerX, paddingXY 4 8, spacing 8 ])
        [ homeInput
        , Element.el [ Font.color Color.grey, UI.Font.mono, centerY ] (Element.text "-")
        , awayInput
        ]
```

If the existing `viewInput` function is used elsewhere in the file, keep it or update call sites to use the split functions. If it is only called from the `view` composition block, replace it entirely with the two functions above.
