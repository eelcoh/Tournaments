<objective>
Make the outer page padding responsive so group match rows fit on a 320px-wide screen (iPhone SE) without horizontal overflow.

Purpose: The root page layout in `View.elm` uses 24px padding on both sides at all screen sizes. The inner content wrapper adds another 24px. On a 320px screen this leaves only 224px for content, but the group match `viewInput` row needs ~228px ŌĆö causing horizontal overflow. Reducing padding to 8px on Phone gives 296px of content space, comfortably fitting all content.

Output: Modified `src/View.elm` using `Screen.device model.screen` to select 8px padding for Phone and 24px for Computer, applied to both the page `paddingEach` and the inner contents `el`.
</objective>

In `src/View.elm`, inside the `view` function, make both the inner `contents` el padding and the outer `page` column `paddingEach` responsive based on `Screen.device model.screen`.

**Change 1: Inner contents wrapper**

Find the `contents` let binding in `view`:
```elm
-- BEFORE
contents =
    let
        contents_ =
            case model.app of
                ...
    in
    Element.el [ Element.padding 24 ] contents_
```

Change to:
```elm
-- AFTER
contents =
    let
        contents_ =
            case model.app of
                ...

        contentPadding =
            case Screen.device model.screen of
                Screen.Phone ->
                    Element.padding 8

                Screen.Computer ->
                    Element.padding 24
    in
    Element.el [ contentPadding ] contents_
```

**Change 2: Page column paddingEach**

Find the `page` let binding:
```elm
-- BEFORE
page =
    Element.column
        [ Element.paddingEach { top = 24, right = 24, bottom = 40, left = 24 }
        , Element.spacing 24
        , Element.centerX
        ]
        [ links
        , contents
        , viewVersion
        ]
```

Change to:
```elm
-- AFTER
page =
    let
        hPad =
            case Screen.device model.screen of
                Screen.Phone ->
                    8

                Screen.Computer ->
                    24
    in
    Element.column
        [ Element.paddingEach { top = 24, right = hPad, bottom = 40, left = hPad }
        , Element.spacing 24
        , Element.centerX
        ]
        [ links
        , contents
        , viewVersion
        ]
```

`Screen` is already imported in `View.elm` (`import UI.Screen as Screen`). `Screen.device` and `Screen.Phone` / `Screen.Computer` are already exposed by `UI.Screen` module. No new imports needed.
