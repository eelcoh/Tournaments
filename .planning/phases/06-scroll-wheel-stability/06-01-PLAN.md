Rewrite the group matches scroll wheel so the active match is always at line 4 of a fixed 7-line window, with the group label anchored at line 1, blank padding lines filling empty positions, and an END marker that never appears above line 4.

Purpose: The current implementation slices 5 match items and adds group boundaries dynamically, producing a variable-length window where the active match drifts in position and height jumps occur at group transitions. This breaks SCRW-01, SCRW-02, and SCRW-03.

Output: A rewritten `viewScrollWheel` function in `src/Form/GroupMatches.elm` that always renders exactly 7 lines, active match at line 4, group label pinned at line 1, no height shifts.
</objective>

New `buildWindow` function — replaces `buildScrollItems`:

Function signature:
```elm
buildWindow : MatchID -> List ( MatchID, AnswerGroupMatch ) -> List WindowLine
```

Algorithm:
1. Build the full flat scroll sequence: call `buildScrollItems` on all 48 matches, convert each `ScrollItem` to `WindowLine` (`MatchRow m -> WLMatch m`, `GroupBoundary g -> WLGroupLabel g`), then append `WLEndMarker` at the end.
2. Find the index of the `WLMatch` item whose MatchID equals the cursor in this full flat list. Use `List.Extra.findIndex`.
3. Extract 3 items BELOW the active match (indices cursorIdx+1, +2, +3) — pad with `WLPadding` if fewer than 3 items exist below. Use `List.Extra.getAt` for each index.
4. Extract 3 items ABOVE the active match (indices cursorIdx-3, cursorIdx-2, cursorIdx-1) — pad with `WLPadding` if fewer than 3 items exist above. Order: `above = [item at cursorIdx-3, item at cursorIdx-2, item at cursorIdx-1]` so that above[0] is line 1 and above[2] is line 3.
5. Group label anchoring (SCRW-02): Find the active match's group via `groupOfMatch`. Check if `above[0]` (line 1) is already `WLGroupLabel activeGroup`. If yes, leave as-is. If no, replace `above[0]` with `WLGroupLabel activeGroup`. Never touch `above[1]` or `above[2]`.
6. Assemble exactly 7 items: `[ above[0], above[1], above[2], WLMatch cursorEntry, below[0], below[1], below[2] ]`
7. Return this list of exactly 7 `WindowLine` items.

Why SCRW-03 is satisfied automatically: `WLEndMarker` is appended after all matches in the full sequence. It can only ever appear in the `below` slice (indices greater than cursorIdx). It can never end up in the `above` slice or at the cursor position (line 4). So it will only appear at lines 5, 6, or 7.

Why SCRW-01 is satisfied: The cursor match is always placed at index 3 (line 4) by construction in step 6.

Blank padding rendering: `WLPadding` renders as `Element.el [ Element.height (Element.px 44) ] Element.none` — invisible but takes up exactly 44px, matching match row height.

New `viewWindowLine` function:
```elm
viewWindowLine : MatchID -> WindowLine -> Element.Element Msg
viewWindowLine cursor line =
    case line of
        WLMatch matchData ->
            viewScrollLine cursor matchData

        WLGroupLabel grp ->
            Element.el
                [ centerX
                , Font.color Color.grey
                , UI.Font.mono
                , Element.height (Element.px 44)
                , centerY
                ]
                (Element.text ("-- " ++ G.toString grp ++ " --"))

        WLPadding ->
            Element.el [ Element.height (Element.px 44) ] Element.none

        WLEndMarker ->
            Element.el
                [ centerX
                , Font.color Color.grey
                , UI.Font.mono
                , Element.height (Element.px 44)
                , centerY
                ]
                (Element.text "-- END --")
```
